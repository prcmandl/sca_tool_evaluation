Definitions and column semantics for the per-ecosystem statistics table:

Ecosystem
  Identifier of the package ecosystem (e.g., pypi, npm, maven, nuget).

Components
  Number of unique components in the final dataset for this ecosystem.
  A component is defined as a unique tuple (component_name, component_version).

OSV-Vulns
  Number of OSV vulnerability entries associated with this ecosystem.
  Each entry corresponds to one OSV advisory affecting a specific component version.

CVE-Findings
  Number of CVE-backed vulnerability findings.
  A CVE-backed finding is defined as a unique tuple
  (component_name, component_version, CVE-ID).
  Multiple OSV advisories may reference the same CVE and are deduplicated here.

CVEs
  Number of unique CVE identifiers referenced by OSV advisories
  in this ecosystem, independent of affected components or versions.

Comp/OSV
  Ratio of unique components to OSV vulnerability entries for this ecosystem:
    Comp/OSV = Components / OSV-Vulns
  This metric indicates component diversity relative to the number of vulnerabilities.

CVE/OSV
  Ratio of CVE-backed findings to OSV vulnerability entries:
    CVE/OSV = CVE-Findings / OSV-Vulns
  Values below 1 indicate that multiple OSV advisories map to the same CVE.
  This ratio characterizes the degree of advisory-to-CVE consolidation.

Vuln-Share
  Proportion of OSV vulnerability entries of this ecosystem relative
  to the total number of OSV entries across all ecosystems:
    Vuln-Share = OSV-Vulns_ecosystem / OSV-Vulns_global

Comp-Share
  Proportion of unique components of this ecosystem relative
  to the total number of unique components across all ecosystems:
    Comp-Share = Components_ecosystem / Components_global

Component frequency statistics (post-balancing):

The following columns characterize how often individual components and
component-version pairs occur in the final ground-truth dataset. Frequency
is defined as the number of OSV vulnerability entries associated with the
respective entity.


MaxCompFreq
  Maximum number of vulnerability entries associated with any single
  component (identified by component_name) within the ecosystem.

MaxCompVerFreq
  Maximum number of vulnerability entries associated with any single
  component-version pair (component_name, component_version).

AvgCompFreq
  Average number of vulnerability entries per component
  (component_name), computed over all components in the ecosystem.

AvgCompVerFreq
  Average number of vulnerability entries per component-version pair
  (component_name, component_version).

MinCompFreq
  Minimum number of vulnerability entries associated with any component
  in the ecosystem.

MinCompVerFreq
  Minimum number of vulnerability entries associated with any
  component-version pair.

MedianCompFreq
  Median number of vulnerability entries per component, providing a
  robust central tendency measure that is less sensitive to outliers.

MedianCompVerFreq
  Median number of vulnerability entries per component-version pair,
  reflecting the typical vulnerability density at version granularity.


TOTAL
  Aggregated sums across all ecosystems.
  Ratio columns in the TOTAL row are computed from the summed values
  and therefore reflect global averages.
Invocation parameter semantics:

SAMPLES_PER_ECOSYSTEM
  Number of components sampled per ecosystem prior to OSV querying.

ECOSYSTEMS
  Space-separated list of ecosystems included in the dataset construction.

START_DATE
  Optional lower bound on component release dates.

END_DATE
  Optional upper bound on component release dates.

BALANCE_ENABLED
  Indicates whether post-hoc balancing across ecosystems is applied.

BALANCE_STRATEGY
  Strategy used to determine the per-ecosystem balancing target (min or median).

MIN_UNIQUE_COMPONENT_RATIO
  Minimum required ratio of unique components during balancing.

BALANCE_SEED
  Random seed used for deterministic post-hoc balancing.

Parameter-specific interpretation:

PYPI_MAX_VERSIONS_PER_PACKAGE
  Maximum number of released versions considered per PyPI package.

NPM_MAX_VERSIONS_PER_PACKAGE
  Maximum number of released versions considered per npm package.

MAVEN_MAX_VERSIONS_PER_PACKAGE
  Maximum number of released versions considered per Maven artifact.

NUGET_MAX_VERSIONS_PER_PACKAGE
  Maximum number of released versions considered per NuGet package.

NUGET_MAX_VERSIONS_PER_PACKAGE
  Maximum number of released versions considered per NuGet package.

A component is defined as a unique tuple (ecosystem, component_name, component_version).
Each OSV vulnerability entry is counted uniquely by (ecosystem, component, version, vulnerability_id).
CVEs are treated as aliases of OSV vulnerabilities and are deduplicated only for CVE-based metrics.
Pre-balancing coverage reports the raw OSV results after canonicalization and before validation and balancing.
Post-hoc balancing approximates equal vulnerability coverage across ecosystems while maximizing component diversity.
If diversity constraints are infeasible, a soft fallback reduces the per-ecosystem target accordingly.
